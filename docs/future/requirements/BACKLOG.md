## Epic 1: [EPIC-001] 用户认证与体验

**描述**：提供安全的用户认证系统和独特的品牌化交互动画体验

**业务价值**：建立用户信任，塑造品牌差异化形象

---

### Feature: [FEAT-001] 用户注册登录系统

**描述**：完整的 JWT-based 用户认证系统

#### US-001: 用户注册
- **作为** 网站访客
- **我想要** 使用邮箱和密码注册账户
- **以便于** 成为平台会员，使用更多功能
- **优先级**: M
- **验收标准**:
  - [ ] 输入有效邮箱格式和密码（8位以上，含大小写和数字）
  - [ ] 两次密码输入一致
  - [ ] 注册成功后自动登录并跳转首页
  - [ ] 邮箱唯一性校验，重复注册提示错误
  - [ ] 密码加密存储（bcrypt）
- **估算**: M

#### US-002: 用户登录
- **作为** 已注册用户
- **我想要** 使用邮箱和密码登录
- **以便于** 访问我的个人内容和设置
- **优先级**: M
- **验收标准**:
  - [ ] 正确的邮箱密码组合登录成功
  - [ ] 错误的凭证给出明确错误提示（不泄露具体哪个字段错误）
  - [ ] 登录成功返回 JWT Access Token 和 Refresh Token
  - [ ] 登录状态持久化（可选择"记住我"）
- **估算**: M

#### US-003: JWT Token 管理
- **作为** 系统
- **我想要** 实施安全的 Token 管理策略
- **以便于** 保障用户会话安全
- **优先级**: M
- **验收标准**:
  - [ ] Access Token 有效期 15 分钟
  - [ ] Refresh Token 有效期 7 天
  - [ ] Refresh Token 存储在 HttpOnly Cookie
  - [ ] Token 被盗用可被检测和撤销
  - [ ] 支持 token 刷新接口
- **估算**: M

#### US-004: 会话保持
- **作为** 已登录用户
- **我想要** 关闭浏览器后重新打开仍然保持登录状态
- **以便于** 无需每次都输入密码
- **优先级**: S
- **验收标准**:
  - [ ] 勾选"记住我"后，7天内无需重新登录
  - [ ] 未勾选则关闭浏览器后会话失效
  - [ ] 支持多设备登录管理
- **估算**: S

### Feature: [FEAT-002] 3D 拟人化球体交互动画 ⭐

**描述**：品牌化 3D 拟人化球体，提供独特的交互体验

**差异化价值**：这是本项目的核心亮点，通过拟人化交互创造情感连接

#### US-005: 球体渲染与基础动画
- **作为** 网站访客
- **我想要** 看到一个可爱的拟人化 3D 球体
- **以便于** 获得独特的第一印象
- **优先级**: M
- **验收标准**:
  - [ ] 使用 React Three Fiber 渲染 3D 球体
  - [ ] 球体材质逼真，有光照和阴影
  - [ ] 球体有呼吸动画（微微起伏）
  - [ ] 页面加载时球体从远处飞入
  - [ ] 帧率保持 60fps
- **估算**: L

#### US-006: 眼睛跟随鼠标移动
- **作为** 网站访客
- **我想要** 看到球体的眼睛跟随我的鼠标移动
- **以便于** 感受到球体在"注意"我
- **优先级**: M
- **验收标准**:
  - [ ] 眼球能跟随鼠标在屏幕范围内移动
  - [ ] 眼球旋转角度有限制，不会翻转
  - [ ] 跟随动作平滑，无跳跃
  - [ ] 鼠标移出窗口时眼球回到中心
- **估算**: M

#### US-007: 点击密码框捂住眼睛
- **作为** 网站访客
- **我想要** 当我点击密码框时，球体用手捂住眼睛
- **以便于** 感受到有趣的隐私保护隐喻
- **优先级**: M
- **验收标准**:
  - [ ] 点击任意密码输入框触发捂眼动画
  - [ ] 手掌从球体两侧伸出并遮住眼睛
  - [ ] 动画时长 300ms，ease-out 缓动
  - [ ] 移开焦点后手掌收回
  - [ ] 支持多个密码框联动
- **估算**: L

#### US-008: 表情状态变化
- **作为** 网站访客
- **我想要** 看到球体根据交互状态变化表情
- **以便于** 获得更丰富的情感反馈
- **优先级**: S
- **验收标准**:
  - [ ] 默认状态：微笑，眼睛有高光
  - [ ] 鼠标悬停：眨眼动画
  - [ ] 密码框聚焦：害羞表情（脸颊泛红）
  - [ ] 登录成功：开心大笑动画
  - [ ] 登录失败：失落表情
  - [ ] 表情切换平滑过渡
- **估算**: M

#### US-009: 球体响应式适配
- **作为** 使用移动设备的访客
- **我想要** 在手机上也能看到球体动画
- **以便于** 获得一致的品牌体验
- **优先级**: S
- **验收标准**:
  - [ ] 移动端球体尺寸自适应
  - [ ] 触摸替代鼠标跟随
  - [ ] 捂眼动画通过触摸触发
  - [ ] 低性能设备降级处理
- **估算**: M

### Feature: [FEAT-004] OpenClaw 接入 ⭐

**描述**：支持 OpenClaw 等 AI 智能体通过标准化接口接入内容

**差异化价值**：Agents 友好型平台的核心能力

#### US-015: Webhook 接收端点
- **作为** OpenClaw 智能体
- **我想要** 通过 Webhook 实时推送内容
- **以便于** 将爬取的 AI 资讯自动同步到平台
- **优先级**: M
- **验收标准**:
  - [ ] POST /api/v1/ingest/webhook 端点可访问
  - [ ] 支持 JSON payload 格式
  - [ ] 验证 Webhook 签名（HMAC-SHA256）
  - [ ] 返回接收确认（200 OK）
  - [ ] 记录接收日志
  - [ ] 支持重试机制（3次重试，指数退避）
- **估算**: M

#### US-016: Feed 批量导入
- **作为** 内容提供商
- **我想要** 批量导入 RSS/Atom/JSON Feed
- **以便于** 高效接入大量历史内容
- **优先级**: M
- **验收标准**:
  - [ ] POST /api/v1/ingest/feed 端点
  - [ ] 支持 RSS 2.0、Atom 1.0、JSON Feed 1.1
  - [ ] 自动解析并存储文章
  - [ ] 去重处理（基于 URL 或标题 hash）
  - [ ] 返回导入报告（成功数、失败数、跳过数）
- **估算**: M

#### US-017: 接入源管理
- **作为** 管理员
- **我想要** 管理所有内容接入源
- **以便于** 监控和维护数据源
- **优先级**: S
- **验收标准**:
  - [ ] CRUD 接入源配置
  - [ ] 显示每个源的最后同步时间
  - [ ] 显示每个源的文章数量
  - [ ] 手动触发同步
  - [ ] 启用/禁用接入源
- **估算**: M

#### US-018: 接入日志查看
- **作为** 管理员
- **我想要** 查看内容接入日志
- **以便于** 排查接入问题和监控数据质量
- **优先级**: S
- **验收标准**:
  - [ ] 记录每次接入的时间、来源、状态
  - [ ] 记录失败原因和错误详情
  - [ ] 支持按来源、时间、状态筛选
  - [ ] 日志保留 30 天
- **估算**: S

#### US-019: OpenClaw Agent 认证
- **作为** OpenClaw 智能体
- **我想要** 通过 API Key 进行身份认证
- **以便于** 安全地接入平台服务
- **优先级**: M
- **验收标准**:
  - [ ] 为每个接入源生成唯一的 API Key
  - [ ] API Key 支持撤销和轮换
  - [ ] 认证失败返回 401
  - [ ] 支持设置 API Key 的权限范围
- **估算**: M
